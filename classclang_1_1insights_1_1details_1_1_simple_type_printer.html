Detailed Description

SimpleTypePrinter a partially substitution of Clang's TypePrinter.

With Clang 9 there seems to be a change in lib/AST/TypePrinter.cpp in printTemplateTypeParmBefore. It now inserts auto when it is a lambda auto parameter. Which is correct, but C++ Insights needs the template parameter name to make up compiling code. Hence, this if "overrides" the implementation of TypePrinter in that case.

It also drops some former code which handled ClassTemplateSpecializationDecl in a special way. Here the template parameters can be lambdas. Those they need a proper name.
