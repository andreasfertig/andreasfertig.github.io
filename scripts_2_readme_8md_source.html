5 ## docker-shell.sh
6 
7 For local build you can use `docker-shell.sh`. It uses a pre-built Docker image from DockerHub
8 [andreasfertig/cppinsights-builder-gitpod](https://cloud.docker.com/repository/docker/andreasfertig/cppinsights-builder-gitpod).
9 This is also the same image which is used by [GitPod](https://www.gitpod.io) and a addition to the
10 [cppinsights-builder](https://cloud.docker.com/repository/docker/andreasfertig/cppinsights-builder) image which is used
11 by [Travis CI](https://travis-ci.org/) for the continuous integration builds. If you execute the script from within the
12 `script` folder it automatically mounts the C++ Insights repo in the Docker to `/workspace/cppinsights`. You also need
13 to specify a build folder which is inside the container mounted to `/workspace/build`.
14 
15 The script can be invoked with two different modes:
16 * `shell` launches a `bash` in the Docker container. This allows to use `gdb` for example.
17 * `compile` does a complete C++ Insights build. When using compile you can provide two additional arguments.
18  * A string with CMake flags used to configure the build
19  * and optional target to invoke. For example, `coverage` to run a code coverage build.
20 
21 Some examples command line looks like this:
22 
23 ```
24 cd scripts
25 ./docker-shell.sh /home/username/build/docker_insights compile "-DINSIGHTS_STATIC=Yes -DINSIGHTS_COVERAGE=Yes -DDEBUG=Yes" coverage
26 ```
27 
28 it triggers a CMake build with code coverage turned on and used as the `ninja` target.
29 
30 ```
31 cd scripts
32 ./docker-shell.sh /home/username/build/docker_insights shell
33 ```
34 
35 Opens a shell into the Docker container.

38 ### CMake and Docker
39 
40 There are some CMake target which use Docker and the `docker-shell.sh` script. The results can be found in the binary folder you supplied plus
41 `docker_build`,
42 
43 * `docker-build` to run a build in Docker.
44 * `docker-build-run` gives you a shell in Docker (same as invoking `docker-shell.sh` with `shell`).
45 * `docker-tests` to run the tests in Docker
46 * `docker-coverage` to run a code coverage build in Docker.
